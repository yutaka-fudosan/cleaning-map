# 清掃物件MAP

Googleスプレッドシート「清掃巡回管理」の「物件一覧」シートから、清掃対象物件を取得して表示する独立したWebマップです。地図にはLeafletとOpenStreetMapを使用し、地図用のAPIキーは不要です。

## ファイル構成

- `index.html`：MAP画面の構造
- `style.css`：パソコン・スマートフォン向けのデザイン
- `app.js`：地図表示、データ取得、検索、絞り込み、現在地表示
- `apps-script/Code.gs`：Google Apps Scriptへ貼り付けるコード
- `README.md`：設定・公開手順

`properties.json`は使用しません。物件情報はGoogle Apps ScriptのWebアプリから取得します。

## 「物件一覧」シートの準備

「清掃巡回管理」スプレッドシートに、シート名が正確に `物件一覧` となっているシートを用意します。

1行目には次の見出しを入力してください。列の順番は自由です。

- 物件名
- 住所
- 物件種別
- 清掃回数
- 緯度
- 経度
- 注意事項

入力ルール：

- 物件名が空欄の行はMAPへ送信されません。
- 物件種別は `アパート` または `駐車場` と入力します。
- 清掃回数は `月1回` または `月2回` と入力します。
- 緯度と経度は数値で入力します。
- 緯度または経度が空欄・数値以外の行は、MAPへ送信されません。
- 物件名と同じ名前のシートがある場合、そのシートを直接開くリンクがMAPに自動表示されます。
- 同名の物件シートがない場合は、MAPに「物件シートなし」と表示されます。

## Apps Scriptの設定

1. Googleスプレッドシート「清掃巡回管理」を開きます。
2. 上部メニューから「拡張機能」→「Apps Script」を開きます。
3. Apps Scriptのエディタで `Code.gs` を開き、既存の内容を削除します。
4. このプロジェクトの `apps-script/Code.gs` の内容をすべてコピーして貼り付けます。
5. プロジェクトを保存します。

このApps Scriptはスプレッドシートに紐づけて使用します。スプレッドシートIDは `getActiveSpreadsheet()` で取得するため、コードへのID入力は不要です。

## Webアプリとして公開する手順

1. Apps Script画面右上の「デプロイ」→「新しいデプロイ」を選択します。
2. 「種類の選択」で「ウェブアプリ」を選択します。
3. 「次のユーザーとして実行」は自分を選択します。
4. 「アクセスできるユーザー」は、MAPからアクセスできる範囲を選択します。公開MAPからログインなしで読み込む場合は「全員」を選択してください。
5. 「デプロイ」を押します。
6. 初回は権限確認画面が表示されるため、内容を確認して許可します。
7. 発行されたウェブアプリURLをコピーします。テスト用URLではなく、末尾が `/exec` のURLを使用してください。

コードを更新した場合は、「デプロイ」→「デプロイを管理」から既存デプロイを編集し、新しいバージョンとして再デプロイしてください。

## MAPへWebアプリURLを設定する

`app.js` 冒頭の次の行を開きます。

```javascript
const DATA_ENDPOINT = "ここにApps ScriptのWebアプリURLを入力";
```

引用符の中を、コピーしたウェブアプリURLへ置き換えます。

```javascript
const DATA_ENDPOINT = "https://script.google.com/macros/s/発行されたID/exec";
```

URLが未設定の場合、MAPには「Googleスプレッドシートとの接続が設定されていません」と表示されます。通信やApps Script側の設定に問題がある場合は「清掃物件データを読み込めませんでした」と表示されます。

## MAPの公開と確認

`index.html`、`style.css`、`app.js`を同じ階層に置いてWebサーバーへ公開します。GitHub Pagesを利用する場合も、この3ファイルを公開対象に含めてください。

ローカル確認では、`index.html`を直接ダブルクリックせず、Webサーバー経由で開いてください。

```powershell
python -m http.server 8000
```

ブラウザで `http://localhost:8000` を開きます。

## 注意事項

- Apps ScriptのウェブアプリURLや公開範囲を変更した場合は、MAP側の設定も確認してください。
- スプレッドシートや物件別シートの閲覧権限は、実際にMAPを利用する人が開ける設定にしてください。
- OpenStreetMap、Googleマップ、Googleスプレッドシートへのアクセスにはインターネット接続が必要です。
